# Web 端设计文档

**项目**: PA Trading Assistant  
**日期**: 2026-05-29  
**状态**: 已批准

---

## 1. 概述

为 Price Action Trading Assistant 构建 Web 界面，将 CLI 分析能力可视化，支持交互式参数调整和历史数据回放。

### 1.1 目标

- **交互式分析工具**：可调整参数、切换时间周期、对比不同分析模块
- **复盘回测平台**：历史数据回放，逐根 K 线推进，验证交易策略

### 1.2 非目标

- 实时 WebSocket 行情推送（后续阶段）
- 自动交易/下单功能
- 多用户/权限管理

## 2. 技术栈

| 层级 | 技术 | 理由 |
|------|------|------|
| 后端 | FastAPI | 项目已有可选依赖，Python 生态无缝集成 |
| 前端交互 | HTMX | 轻量级，无需构建工具，服务端渲染 |
| 模板引擎 | Jinja2 | FastAPI 原生支持 |
| 图表库 | Lightweight Charts (TradingView) | 开源 K 线图库，支持叠加标注 |
| 样式 | Tailwind CSS (CDN) | 快速开发，深色主题友好 |
| 数据源 | DuckDB (现有) | 复用现有数据层，无需额外数据库 |

## 3. 设计系统

基于 ui-ux-pro-max 推荐的 Fintech/Crypto 设计方案。

### 3.1 颜色

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色 | `#F59E0B` (金色) | 品牌色，CTA，强调 |
| 次色 | `#FBBF24` | 辅助强调 |
| 背景 | `#0F172A` (Slate 900) | 主背景 |
| 卡片 | `#1E293B` (Slate 800) | 卡片/面板背景 |
| 边框 | `#334155` (Slate 700) | 分隔线/边框 |
| 文本 | `#F8FAFC` (Slate 50) | 主文本 |
| 次文本 | `#94A3B8` (Slate 400) | 辅助信息 |
| 看涨 | `#22C55E` (Green 500) | 上涨/看涨信号 |
| 看跌 | `#EF4444` (Red 500) | 下跌/看跌信号 |
| 警告 | `#F59E0B` | Stop Hunt/需关注 |

### 3.2 字体

- **标题**: Orbitron (Google Fonts) - 科技感
- **正文**: Exo 2 (Google Fonts) - 可读性好

### 3.3 设计原则

- 深色 OLED 主题，减少眼疲劳
- 高对比度确保数据可读性
- 无 Emoji 图标，使用 SVG (Lucide Icons)
- 所有可点击元素添加 `cursor-pointer`
- 响应式：375px / 768px / 1024px / 1440px

## 4. 页面设计

### 4.1 主仪表盘 (`/`)

**布局**: 三栏布局

```
┌─────────────────────────────────────────────────────────────┐
│  [Logo]  BTCUSDT  $67,234    [仪表盘] [流动性] [回测] [告警] │
├──────────┬──────────────────────────────┬───────────────────┤
│          │                              │                   │
│  参数    │       K 线图                 │    分析结果       │
│  面板    │    + 分析叠加                │                   │
│          │                              │  ┌─────────────┐  │
│ [时间周期]│  [OB] [FVG] [流动性] [BOS]  │  │  Wyckoff    │  │
│ [Lookback]│                              │  │  累积 B     │  │
│ [容差]   │                              │  └─────────────┘  │
│ [模块开关]│                              │  ┌─────────────┐  │
│          │                              │  │  趋势       │  │
│ [运行]   │                              │  │  ↑ 上升     │  │
│          │                              │  └─────────────┘  │
│          │                              │  ┌─────────────┐  │
│          │                              │  │  流动性池   │  │
│          │                              │  └─────────────┘  │
└──────────┴──────────────────────────────┴───────────────────┘
```

**左侧参数面板**:
- 时间周期选择器: 5m / 15m / 1h / 4h / 1d
- HTF 周期 (可选): 无 / 4h / 1d
- Swing Lookback: 滑块 1-5
- 流动性池容差: 滑块 1-20 bps
- Volume Climax Z: 滑块 1.0-4.0
- 分析模块开关: 复选框组
- 运行分析按钮

**中间 K 线图**:
- 使用 Lightweight Charts 渲染
- 叠加层: OB (半透明矩形) / FVG (虚线矩形) / 流动性池 (水平虚线) / BOS/CHoCH (垂直虚线)
- 工具栏: 十字线 / 测量 / 截图 / 缩放

**右侧分析结果**:
- Wyckoff 状态卡片 (阶段/区间/置信度/下一步)
- 趋势卡片 (工作周期 + HTF)
- 流动性池列表 (上方/下方，距离排序)
- 背离列表 (指标/方向/强度)
- 资金费率/OI 摘要

**交互流程**:
1. 页面加载时自动运行最近数据的分析
2. 用户调整参数后点击「运行分析」或自动触发
3. HTMX 请求 `/api/analyze` 获取分析结果
4. 前端更新图表叠加层和右侧结果卡片

   - 进度条拖拽

2. **区间快照**:
   - 选择起止时间
   - 一次性生成该时段的分析报告
   - 可导出为 Markdown/JSON

**WebSocket 推送**:
- 回放模式下使用 WebSocket 逐根推送数据
- 前端实时更新图表和分析结果
- 支持暂停/恢复

## 5. API 设计

### 5.1 分析 API

```
POST /api/analyze
```

**请求参数**:
```json
{
  "symbol": "BTCUSDT",
  "timeframe": "1h",
  "htf": "4h",
  "swing_lookback": 3,
  "eq_tolerance_bps": 10.0,
  "volume_climax_z": 2.0,
  "modules": ["structure", "zones", "liquidity", "wyckoff", "divergence"]
}
```

**响应**:
```json
{
  "timestamp": "2026-05-29T12:00:00Z",
  "symbol": "BTCUSDT",
  "timeframe": "1h",
  "current_price": 67234.50,
  "wyckoff": {
    "phase": "accumulation_phase_b",
    "confidence": 0.67,
    "range_low": 66500,
    "range_high": 68200,
    "events": [...]
  },
  "trend": {
    "working": "up",
    "htf": "up",
    "alignment": "bullish"
  },
  "liquidity_levels": [...],
  "order_blocks": [...],
  "fvgs": [...],
  "structure_events": [...],
  "divergences": [...],
  "scorecard": {
    "net_bias": "bullish",
    "bullish_factors": [...],
    "bearish_factors": [...]
  }
}
```

### 5.2 回放 API

```
WebSocket /ws/replay
```

**连接参数**:
```
ws://localhost:8000/ws/replay?symbol=BTCUSDT&timeframe=1h&start=2026-05-01T00:00:00Z&speed=3
```

**服务端推送消息**:
```json
{
  "type": "bar",
  "timestamp": "2026-05-01T01:00:00Z",
  "bar_index": 1,
  "ohlcv": {
    "open": 67000,
    "high": 67200,
    "low": 66800,
    "close": 67100,
    "volume": 1234.56
  },
  "analysis": {
    "wyckoff_phase": "neutral",
    "structure_events": [],
    "active_obs": [],
    "active_fvgs": [],
    "liquidity_levels": [...]
  }
}
```

**客户端控制消息**:
```json
{"type": "pause"}
{"type": "resume"}
{"type": "step_forward"}
{"type": "step_backward"}
{"type": "seek", "bar_index": 100}
{"type": "set_speed", "speed": 5}
```

## 9. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Lightweight Charts 叠加层复杂 | 高 | 先实现基础 K 线，逐步添加叠加层 |
| WebSocket 回放内存占用 | 中 | 限制历史窗口大小，使用生成器 |
| DuckDB 并发读取性能 | 中 | 使用连接池，添加缓存层 |
| 深色主题对比度 | 低 | 使用 ui-ux-pro-max 推荐的配色方案 |
